AWS의 퍼블릭 클라우드에 생성된 리소스들과 내외부 간의 통신에 관련된 기능을 제공함
예) VPC, CloudFront, Route53

Virtual Private Cloud(VPC)
- AWS 퍼블릭 클라우드 내에 구성한 독립적인 클라우드 네트워크
- VPC는 물리적 개념이 아니라 논리적 개념임
- AWS의 네트워크 인프라를 활용해 구성한 논리적이고 독립적인 네트워크임
- VPC는 특정 리전에 속함
- 즉, 같은 리전 내의 리소스들에 한해서만 독립적인 네트워크를 구성할 수 있음
- 한 리전 내에 여러 VPC를 생성할 수 있으며, 각 VPC는 서로 독립적임
- 다른 리전의 VPC와 서로 연결하여 클라우드 네트워크를 확장할 수도 있음

서브넷과 가용 영역
- 서브넷은 가용 영역과 1:1 관계임
- 하나의 서브넷은 하나의 가용 영역에 할당됨

서브넷의 분류
- 퍼블릭 서브넷
1) VPC 내부 및 외부 인터넷 구간 모두 연결되어 있음
2) VPC 내부 리소스 및 외부 인터넷과 통신 가능함
- 프라이빗 서브넷
1) VPC 내부 리소스들끼리만 연결되어 있음
2) VPC 내부 리소스와 통신 가능, 외부 인터넷과 통신 불가함

VPC와 서브넷의 IP
- VPC와 서브넷 모두 IP를 CIDR 방식으로 표기함
- VPC IP 10.0.0.0/16
- 서브넷 마스크가 16 : 2번째 옥텟까지는 네트워크 주소임
- 3번째 옥텟부터 IP 주소 할당 가능함
- 2^16 - 2 = 65,534개
- 서브넷_a IP 10.0.1.0/24
- 서브넷 마스크가 24 : 3번째 옥텟까지는 네트워크 주소임
- 4번째 옥텟부터 IP 주소 할당 가능함
- 2^8 - 2 = 254개

가상 라우터

VPC - 타 네트워크 간 연결
- 퍼블릭 서브넷은 외부 인터넷과 통신 가능함
- 이를 위해서는 인터넷 게이트웨이를 생성해야 함

인터넷 게이트웨이
- VPC내의 리소스가 외부 인터넷과 통신할 수 있도록 연결을 제공하는 네트워크 구성 요소임
- VPC 내의 리소스가 퍼블릭 IP가 할당되어 있을 때 외부 인터넷과 연결될 수 있게 해줌

프라이빗 서브넷은 외부 인터넷과 통신 불가함
- 단, 예외적으로 가능하게 할 수 있음
- NAT 게이트웨이를 생성해야 함

NAT 게이트웨이
- Network Address Translation Gateway
- 프라이빗 서브넷의 프라이빗 IP를 퍼블릭 IP로 전환해줌
- 퍼블릭 서브넷에 위치함

VPC 보안
- VPC는 네트워크 보안을 위해 가상의 방화벽 역할을 하는 두 가지 보안 기능을 제고함

보안 그룹과 네트워크 ACL
네트워크 ACL
- 인바운드 & 아웃바운드 규칙을 개별적으로 설정함
- 서브넷 수준에서 동작함
- 서브넷으로 출입하는 트개픽을 제어함
- 비상태 기반
- 화이트리스트 & 블랙 리스트 기반 순차 평가

보안 그룹
- 인바운드 & 아웃바운드 규칙을 개별적으로 설정함
- 인스턴스 수준에서 동작함
- 인스턴스로 출입하는 트래픽을 제어함
- 상태 기반
- 화이트리스트 기반 동시 평가
