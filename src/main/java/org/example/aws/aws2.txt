AWS 서비스 라인업
- 컴퓨팅 : AWS의 다양한 컴퓨팅 자원을 임대하여 사용할 수 있음
- 스토리지 : AWS의 퍼블릭 클라우드에 데이터를 저장할 수 있음
- 데이터베이스 : AWS가 구축해둔 데이터베이스 시스템을 사용할 수 있음
- 네트워킹 및 콘텐츠 전송 : AWS의 퍼블릭 클라우드에 생성된 리소스들과 내외부 간의 통신에 관련된 기능을 제공함
- 보안, 자격증명 및 규정준수 : 자격 증명 및 접근 관리, 모니터링 등 리소스와 데이터를 보호하기 위한 도구를 제공함

Elastic Compute Cloud(EC2)
- AWS의 컴퓨팅 자원을 임대하여 가상의 서버를 운영할 수 있는 서비스
- 원하는 만큼 컴퓨팅 자원을 선택하여 임대할 수 있음
- "Elastic" : 사용자의 필요에 따라 서버 자원을 자유롭게 확장 및 축소할 수 있음
- 사용자는 실제로 사용한 컴퓨팅 자원만큼만 비용을 지불함

EC2 인스턴스
- AWS 퍼블릭 클라우드로부터 컴퓨팅 자원을 임대하여 구성한 가상의 컴퓨팅 환경(가상 서버)

인스턴스 유형과 AMI
- 컴퓨팅 자원은 하드웨어 자원과 소프트웨어 자원으로 분류됨
- 따라서 EC2 인스턴스를 생성하려면 어떤 하드웨어 자원과 소프트웨어 자원을 사용할 것인지 선택해야 함
- 인스턴스 유형과 AMI를 통해 각각 하드웨어 자원과 소프트웨어 자원을 지정할 수 있음

인스턴스 유형
- 컴퓨팅 목적에 맞게 AWS가 미리 구성해둔 하드웨어 자원
- 인스턴스 패밀리 : 컴퓨팅 목적 및 용도에 따라 분류됨 (t)2.micro
- 인스턴스 세대 : 성능, 비용 효율성 등이 개선되며 오름차순으로 붙이는 버전임
                높을수록 최근에 출시된 세대를 나타냄 t(2).micro
- 인스턴스 크기 : 가상 CPU, 메모리 용량, 네트워크 성능 등을 나타냄
                nano, micro, small, medium, large, xlarge, 2xlarge, 4xlarge 등 t2.(micro)

인스턴스 유형 - 인스턴스 패밀리
- 범용 : 다양한 워크로드에 사용할 수 있도록 컴퓨팅 자원을 균형 있게 구성
- 컴퓨팅 최적화 : 고성능 프로세서가 필요한 애플리케이션에 최적화된 하드웨어 구성
- 메모리 최적화 : 메모리에서 대규모 데이터셋의 처리가 필요한 워크로드에 최적화된 하드웨어 구성
- 가속화된 컴퓨팅 : GPU 기반의 고성능 컴퓨팅이 필요한 환경에 최적화된 하드웨어 구성
- 스토리지 최적화 : 스토리지에 매우 큰 데이터를 읽거나 쓰는 워크로드에 최적화된 하드웨어 구성
- HPC 최적화 : 고성능 컴퓨팅 워크로드 실행 시 최고의 가격 대비 선능을 낼 수 있도록 조합된 하드웨어 구성

AMI(Amazon Machine Image)
- 인스턴스 구성에 필요한 소프트웨어 컴퓨팅 자원을 명시한 템플릿
- 루트 볼륨 템플릿 : 인스턴스에 사용할 운영 체제 및 기본 소프트웨어 스택 등을 포함
- 설정 파일 및 소프트웨어 : 특정 소프트웨어 패키지와 설정 파일을 포함
- 권한 정보 : 인스턴스 시작 시 필요한 권한 정보를 포함
- 볼륨 스냅샷 : AMI 재사용을 위한 루트 볼륨 및 스토리지 볼륨(EBS)의 특정 시점의 스냅샷을 포함
